# Debugger walkthrough ledgers

Stage 09 records engineer-led, line-by-line debugger walkthrough evidence.

- Reserve the next `DBG` number in `../numbering-registry.md` before creating a ledger.
- Bind every ledger to explicit base and HEAD revisions.
- Assign every changed executable line to exactly one chunk.
- A chunk is complete only after a real breakpoint/trigger and explicit engineer attestation.
- The agent prepares and records; it never self-certifies the walkthrough.
